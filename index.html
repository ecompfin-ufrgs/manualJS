Introdução

Este trabalho objetiva apresentar a linguagem JavaScript, tomando por base o JavaScript Guide e o JavaScript Language Overview ambos da Mozilla Developer Network. Sua escrita pretende-se dar de modo iterativo e incremental. Portanto, este manual não se apresenta de forma acabada, mas em constante evolução.

De iniício, apresentaremos a linguagem propriamente dita a partir de seu documento de especificação. Posteriormente, utilizaremos a linguagem em outros ambientes de interesse, quais sejam, a os navegadores Web, as planilhas Google Sheets e Office Web MS-Excel.

Observe que uma linguagem (humana ou de computador) é um construto usado para um indivíduo - chamado emissor - envie mensagens a outro indivíduo - chamado receptor. Estas mensagens são compostas por sinais que, alocados de modo a respeitar uma regra previamente conhecida pelas partes, formarão termos providos do sentido desejado pelo emissor.

Ao conjunto de sinais ou símbolos disponíveis em uma linguagem para formar termos chama-se léxico. A regra de ordenação dos sinais, dos termos formados por eles e das sentenças formadas por estes termos é chamada de sintaxe.

>

A partir das regras sintáticas pode-se construir axiomaticamente termos e as sentenças que designam objetos do mundo (concreto ou abstrato) conhecidos tanto pelo emissor quanto pelo receptor. Genericamente, chamamos a este processo de construção semântica da linguagem.

Assim, a semântica se divide em regras de oriegem sintáticas para construção dos objetos (atribuição de valores) e sentenças sintaticamente corretas. conjunto dos objetos construídos e das sentenças com sentido (isto é, que efetivamente designam objetos e ações possíveis de serem realizados com estes) de semântica denotacional ou modelo de dados da linguagem. Já as regras de construção chama-se de semântica operacional ou modelo de execução.

Ou seja, uma linguagem é composta pelos três seguintes elementos:

  • semântica;
  • léxico, e
  • sintaxe (denotacional e operacional).

O léxico da linguagem JavaScript é formado pelo conjunto de caracteres UTF-8 e, por isso, não nos alongaremos na sua exposição. Na páginas seguintes, focaremos nossa atenção na apresentação da sintaxe e da semântica da linguagem, isto é, a sintaxe para construir os objetos da linguagem (modelo de dados) acompanhados das operações que podem ser realizadas com eles (modelo de execução).

Paradigmas de programação

>

Um paradigma de programação é um conjunto de regras para organizar o conjunto de instruções dadas ao computador em uma determnada linguagem de programação, isto é, o código de um programa. Constitui-se, então, em um estilo de escrita de código.

A implementação da linguagem JavaScript

A fim de praticar o uso da linguagem, precisamos utilizar uma implementação da linguagem, isto é, um programa que interpreta o código JavaScript, transforma-o em código executável e o executa ou produz um executável. Uma implementação fornece um conjunto de funções para comunicação com o ambiente onde ela está instalada. A este conjunto denominamos de application programming interface (API).

JavaScript foi criada originalmente para automatizar tarefas dentro do navegador Web NetScape Navigator, sendo, posteriormente, adotada como linguagem de script nativa dos principais navegadores Web existentes. Sumariamente, navegadores implementam duas API: API Web e a browser object model) API (BOM API). A API Web oferece funções para manipular objetos que trafegam pela Web, isto é, documentos HTML, folhas de estilo CSS e objetos JavaScript. Ao subconjunto da API Web compostos pelas funções para manipular documentos HTML chamamos de document object model API (DOM API). A BOM API pode depender do fabricante do navegador, porém, na prática, os principais navegadores implementam a mesma BOM API.

Hoje em dia, porém, JavaScript é uma linguagem amplamente utilizada, estando implementada também na suíte de aplicativos Web para escritório Google Docs, no Microsoft Office Web. Também possui duas implementações para execução direta pelo sistemaoperacional de computadores, quais sejam: NodeJS (a primeira e mais usada) e Deno( mais recente).

Naturalmente, diferentes tempos de execução oferecem diferentes API para operar os objetos que têm acesso. Assim, o Node.JS e o Deno acessam objetos da memória livremente ao passo que os navegadores são limitados para garantir a segurança dos usuários contra ataques via Internet.

A operação com a API Web pressupõe o conhecimento dos objetos Web e a BOM API pressupõe o conhecimento dos objetos dos navegadores. Como os objetos componentes dos computadores são mais conhecidos do que os objetos da Web e da BOM, optamos por inicialmente empregar em nossos exemplos iniciais uma implementação no sistema operacional, preferindo o Node.JS por esta ser a mais usada.

Posteriormente, evoluiremos para o uso da API Web e da BOM API, permitindo a construção de interfaces gráficas de usuário a partir da integração de elementos HTML estilizados por seletores CSS.
